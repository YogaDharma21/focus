# Plan: Fix All Overlap Issues (Complete Solution)

## Problem Summary
Previous padding approach is not reliable across all zoom levels. The root issues:
1. **Fixed positioned sidebar** at `left: 24px` doesn't leave enough space on medium/zoomed viewports
2. **Centered content container** conflicts with fixed elements on both sides
3. Padding approach creates uneven layout at different zoom levels
4. Zoom factor changes effective viewport width unpredictably

## Better Solutions (Re-imagined)

### ✅ Recommended Solution: Sidebar Push Layout
**Approach:** Make the sidebar part of the document flow, not fixed position.

**How it works:**
- Sidebar uses normal flow (not fixed) on desktop
- Content container sits next to sidebar naturally
- Media player remains fixed at bottom-right
- No padding hacks required - layout is intrinsically correct

### Alternative Solutions:

#### Option B: Smart Sidebar Hiding
- Sidebar automatically slides off-screen when hovering over content
- Appears on hover or when mouse is near left edge
- Great for maximum screen real estate

#### Option C: Collapsible Sidebar
- Add collapse button to sidebar
- Mini sidebar mode showing only icons (48px width)
- User can toggle between expanded and collapsed states

#### Option D: Bottom Bar Only
- Remove left sidebar entirely
- All navigation stays in bottom bar on all screen sizes
- Simplest solution, most consistent across devices

## New Issue: Task Details Panel & Media Player Overlap

### Problem
When task details sheet is open (right side slide-in) and media player is open, they overlap at the bottom-right corner.

### Root Cause
- Task details sheet: Fixed position, full viewport height, 540px width from right
- Media player: Fixed position, bottom-right corner, 320px width
- Both components use high z-index values, causing visual overlap

## Solution: Context-Aware Bottom Padding

### Implementation Steps

#### 1. **Sheet Component Adjustment** (`components/modules/TodoList.tsx`)
- Add conditional bottom padding to SheetContent when media player is open
- When `mediaPlayerOpen` is true, add `pb-[280px]` to sheet content
- Smooth transition animation for padding change

#### 2. **State Integration**
- Import `useAppStore` in TodoList component
- Access `mediaPlayerOpen` state
- Apply conditional styling to SheetContent

## Recommended Approach: Bottom Padding
✅ **Non-disruptive**: Sheet content simply scrolls above the player
✅ **Smooth**: Transition matches existing animations
✅ **Usable**: All sheet controls remain accessible
✅ **Minimal code change**: Single conditional class
