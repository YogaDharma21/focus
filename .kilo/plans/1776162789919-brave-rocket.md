# Plan: Fix Focus Player Overlap with Main Content

## Problem
The Focus Player (MediaPlayer component) when open overlaps main content areas (Tasks and Journal & Stats views) because:
- Player is fixed position at bottom-right with `z-[100]` (highest layer)
- Main content containers are centered full-width without consideration for player presence
- Player is 320px wide when open, covering content in the bottom-right quadrant

## Selected Solution: Context-Aware Layout Adjustment
**Chosen approach:** Main content shifts right automatically when player is open, preventing overlap while maintaining full functionality of both components.

## Implementation Steps

### 1. **Global Store Integration**
- Locate where `mediaPlayerOpen` state is defined (likely in `lib/store.ts` or similar)
- Ensure this state is properly exported and accessible to the main layout

### 2. **Modify main layout container** (`app/page.tsx`)
- Add conditional right padding: `pr-[344px]` (320px player width + 24px spacing) when `mediaPlayerOpen` is true
- Apply this only on desktop viewports (`lg:` breakpoint and above)
- Add smooth transition animation: `transition-all duration-300 ease-in-out`

### 3. **Verify responsive behavior**
- On mobile (< 1024px): Maintain current behavior (player takes full width or overlays appropriately)
- On desktop: Layout adjusts dynamically when player opens/closes

### 4. **Test all views**
- Verify Tasks view: No overlap with player when open
- Verify Journal & Stats view: No overlap with player when open  
- Verify Focus view: Timer remains centered properly

## Files to Modify:
1. Global state store (where `mediaPlayerOpen` is defined) - if not already exported
2. `app/page.tsx` - Main layout container
3. Optional: `components/modules/MediaPlayer.tsx` - To ensure state updates correctly
